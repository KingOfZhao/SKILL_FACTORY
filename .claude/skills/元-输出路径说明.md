# 元-skill 输出路径说明

## 当前实践

所有元-skills（以"元-"开头的技能）目前在各自目录下存储输出，不使用统一的 `元/` 文件夹结构。

## 推荐结构

```
.claude/skills/
├── skill-figma-html/           # 普通 skill（当前）
├── 元/                             # 统一的输出存储路径（推荐）
│   ├── 元-skill-生成器/
│   │   ├── enumeration/           # 生成的技能
│   │   ├── analysis/              # 检查结果
│   │   └── optimization/          # 优化方案
├── 元-skill-修复器/
│   ├── enumeration/
│   └── analysis/
├── 元-skill-检查器/
│   ├── enumeration/
│   └── analysis/
├── 元-skill-优化器/
│   ├── enumeration/
│   └── analysis/
└── 元-skill-问题穷举器/
│   ├── enumeration/
│   └── analysis/
└── 元-skill-修复器/
│   ├── enumeration/
│   └── analysis/
└── 元-skill-group/               # 元-skills 集合（新建）
└── skills.json                   # 技能列表索引

## 说明

### 为何使用统一路径

1. **当前限制**：各元-skill 是独立项目，维护各自目录更方便
2. **未来升级**：如需要统一输出路径，只需修改每个元-skill 的输出逻辑
3. **灵活性**：不强制所有元-skill 使用相同结构，可根据需要灵活调整

### 实施方式（可选）

如果需要统一为 `元/` 文件夹，可以：

1. 修改元-skill-生成器的输出逻辑，将 `enumeration/` 改为 `元/enumeration`
2. 修改其他元-skills 的输出逻辑指向 `元/[skill-name]/`
3. 更新 元-skill-group 的定义，指定统一的输出路径
4. 更新 underlying-convention.md 的架构解耦规则，添加输出路径规范
