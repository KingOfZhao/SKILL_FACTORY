# 需求冲突检测器（description.md）

## 核心问题
多个需求之间可能存在冲突（功能互斥、依赖循环、资源冲突等），手动检测容易遗漏。

## 问题背景
1. 需求数量多时难以人工检查
2. 隐性冲突容易被忽视
3. 冲突会影响开发进度和质量
4. 需要系统化冲突检测

## 解决方案
自动分析需求列表检测冲突：
- 检测功能互斥（深色 vs 浅色主题）
- 检测依赖循环
- 检测资源冲突（同一资源不同需求）
- 输出冲突报告和解决建议

## 输入
需求列表（JSON 格式）

## 输出
冲突报告：
- conflicts：冲突列表
- type：冲突类型
- severity：严重程度
- resolution_suggestions：解决建议

## 使用场景
1. 需求评审前冲突检测
2. 迭代规划冲突检查
3. 依赖关系验证
4. 需求文档质量保证

## 预期效果
产品经理可以在 10 分钟内：
1. 输入需求列表
2. 获得冲突检测报告
3. 手动验证冲突准确性
4. 决定如何解决冲突

## 已知局限
- 不自动修复冲突
- 依赖需求信息完整性
- 冲突严重度为主观评估
- 解决建议需人工审核
