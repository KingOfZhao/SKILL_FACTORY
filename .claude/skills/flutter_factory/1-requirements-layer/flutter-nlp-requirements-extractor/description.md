# NLP 需求提取器（description.md）

## 核心问题
产品经理和用户使用自然语言描述需求，但开发团队需要结构化的技术规格。手动转换耗时且容易遗漏。

## 问题背景
1. 需求文档通常采用自然语言描述
2. 技术规格提取需要理解功能、约束、平台等多维度信息
3. 人工提取效率低，容易遗漏关键要素
4. 缺乏统一的规格格式标准

## 解决方案
使用 NLP 技术自动解析自然语言需求：
- 识别功能点（用户行为、系统功能）
- 提取平台约束（Android、iOS、Web）
- 提取技术约束（支付方式、API 版本等）
- 输出标准化的 JSON spec

## 输入
自然语言需求文本（纯文本或 Markdown）

## 输出
结构化 JSON spec：
- spec_id：规格编号
- features：功能列表
- platforms：目标平台
- constraints：约束条件
- user_roles：用户角色

## 使用场景
1. 产品需求文档转换为技术规格
2. 用户故事卡片生成
3. 快速原型需求提取
4. 需求文档标准化

## 预期效果
开发团队可以在 10 分钟内：
1. 提取自然语言需求
2. 获得 JSON 格式规格
3. 验证提取结果准确性
4. 直接用于后续开发

## 已知局限
- 不验证需求可行性
- 不检测需求冲突
- 受自然语言清晰度影响
- 依赖 NLP 模型精度
