# 语音需求捕获器（description.md）

## 核心问题
产品经理或用户希望通过语音方式快速录入需求，而不是手动打字。需要将语音转换为文字需求。

## 问题背景
1. 移动时代语音输入更便捷
2. 会议录音需要转换为需求文档
3. 手动打字效率低
4. 语音转文字技术成熟

## 解决方案
使用语音转文字技术捕获需求：
- 支持常见音频格式（.m4a, .mp3, .wav）
- 调用语音转文字 MCP
- 输出转换后的文本和置信度
- 可选结构化提取

## 输入
语音文件（.m4a, .mp3, .wav）

## 输出
语音需求规格：
- transcribed_text：转换后的文本
- confidence_score：置信度
- structured_spec：结构化规格（可选）

## 使用场景
1. 会议录音转需求
2. 语音快速录入需求
3. 离线录音处理
4. 多语言需求录入

## 预期效果
用户可以在 10 分钟内：
1. 录制或提供语音文件
2. 获得文字转换结果
3. 验证转换准确性
4. 获得结构化需求 spec

## 已知局限
- 依赖语音清晰度
- 需要 MCP 服务
- 不支持实时语音
- 嘈杂环境影响精度
