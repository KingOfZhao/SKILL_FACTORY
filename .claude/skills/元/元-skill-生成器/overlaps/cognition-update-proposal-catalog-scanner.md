# 结构化认知更新提案 - Skill 目录扫描器

## 本次生成的新模式/认知

### 1. 目录扫描类 Skill 的标准输出模式

**发现模式**：
- 输出格式应为双格式：Markdown（人类可读）+ JSON（机器可读）
- JSON 结构应包含三个核心部分：
  1. `scan_info`：扫描元数据（路径、时间戳、数量）
  2. `tree_structure`：树状目录结构
  3. `skills_list`：技能详细列表

**模板可复用性**：
此模式可应用于任何需要"扫描目录并生成清单"的场景，如：
- 组件库扫描
- 文档扫描
- 测试文件扫描

### 2. 路径规范化约定

**发现模式**：
- 输出 JSON 中的 `path` 字段应使用相对路径（从项目根目录开始）
- 格式：`.claude/skills/元/元-skill-生成器`
- 便于跨环境使用和路径拼接

### 3. 双重验证指南设计

**发现模式**：
10 分钟验证指南应同时包含：
1. 人类验证步骤（检查 Markdown 可读性）
2. 机器验证步骤（JSON 格式和字段验证）
3. 对比验证步骤（确保扫描准确性）
