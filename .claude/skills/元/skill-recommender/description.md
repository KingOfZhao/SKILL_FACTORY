# 元-Skill推荐器 能力描述

## 核心能力

元-Skill推荐器是一个智能技能推荐引擎，基于问题类型和上下文自动推荐最优的技能组合。

### 智能领域推断

支持 13 个问题领域，通过关键词分析自动识别问题所属领域：

- **bluetooth**: 蓝牙连接、设备扫描、低功耗等蓝牙相关问题
- **rssi**: 信号强度、实时图表、数据可视化等 RSSI 相关问题
- **performance**: 性能优化、卡顿、内存管理、FPS 等性能相关问题
- **algorithm**: 算法、排序、搜索、动态规划等算法问题
- **network**: HTTP 请求、API 调用、网络协议等网络问题
- **database**: SQL 查询、数据库设计、数据管理等数据库问题
- **ui_ux**: 界面设计、布局、组件开发等 UI/UX 问题
- **security**: 认证、加密、Token 管理等安全问题
- **testing**: 单元测试、QA、质量保证等测试问题
- **optimization**: 异步处理、缓存策略、微服务调优等优化问题
- **general**: 其他通用性问题

### 技能推荐机制

#### 基于领域的直接推荐

当技能数据库中存在指定领域的应用层技能时：
1. 直接检索领域相关技能
2. 结合通用工具技能
3. 按相关性排序
4. 生成推荐结果

#### 元技能回退机制

当没有找到特定领域的技能时，自动推荐合适的元技能组合：

| 问题类型 | 推荐的元技能 |
|----------|----------------|
| 优化/性能问题 | 问题穷举器 → 优化器 → 生成器 |
| 生成/实现问题 | 问题穷举器 → 生成器 → 检查器 |
| 检查/验证问题 | 问题穷举器 → 检查器 → 扫描器 |
| 修复问题 | 问题穷举器 → 修复器 → 检查器 |

### 依赖关系分析

- 解析每个技能的依赖关系（从 SKILL.md 提取 "requires" 或 "依赖" 关键词）
- 生成技能依赖关系图
- 检测循环依赖（使用 DFS 算法）
- 在报告中展示依赖关系

### 执行顺序规划

基于以下因素规划技能调用顺序：
1. 技能依赖关系（依赖的技能先执行）
2. 问题类型匹配（优先推荐相关技能）
3. 技能类型优先级（元技能 → 应用技能 → 工具技能）

### 置信度评估

基于以下因素计算推荐置信度（0-1 范围）：
- 技能数量：更多技能 = 更高置信度
- 平均相关性分数：技能与问题的匹配程度

### 执行时间预估

基于技能类型和数量预估执行时间：
- 元技能：权重系数 1.5
- 应用技能：权重系数 1.0
- 工具技能：权重系数 0.5
- 基础时间：每个技能平均 2 分钟

## 输入要求

### 问题描述（必需）
- 类型：自由文本
- 内容：清晰描述需要解决的问题或任务
- 示例："蓝牙连接不稳定，需要实现指数退避和自适应超时策略"

### 问题领域（可选）
- 类型：字符串
- 取值：bluetooth, rssi, performance, algorithm, network, database, ui_ux, security, testing, optimization, general
- 作用：如果不提供，将自动推断

## 输出内容

### Markdown 格式报告
包含以下信息：
- 问题分析（问题描述和推断的领域）
- 推荐技能列表（最多 5 个）
- 每个技能的详细信息（类型、类别、路径、描述、能力）
- 执行顺序
- 推荐理由
- 置信度和预估执行时间
- 依赖关系分析

### JSON 格式导出
机器可读的结构化数据：
- 时间戳
- 推荐列表
- 每个推荐包含：技能详情、推理、置信度、预估时间、执行顺序

## 适用场景

### 场景 1：用户提出 Flutter 开发问题
**输入**："RSSI 图表卡顿，需要实现自适应更新间隔和数据点裁剪"

**推荐**：
1. 元-Skill问题穷举器 - 分析图表性能问题
2. 元-Skill优化器 - 优化更新策略
3. 元-Skill生成器 - 生成实现代码

### 场景 2：用户提出技能优化需求
**输入**："优化蓝牙连接处理，添加指数退避"

**推荐**：
1. 元-Skill问题穷举器 - 穷举连接问题
2. 元-Skill优化器 - 优化连接逻辑
3. 元-Skill生成器 - 生成优化代码
4. 元-Skill检查器 - 验证实现

### 场景 3：用户提出验证需求
**输入**："验证蓝牙扫描器的 10 分钟可验证性"

**推荐**：
1. 元-Skill问题穷举器 - 分析验证需求
2. 元-Skill检查器 - 执行验证检查
3. 元-Skill扫描器 - 扫描能力清单

## 限制条件

1. **技能数据库依赖**
   - 需要预先运行 元-skill-扫描器
   - 生成的 skill-catalog.json 作为技能数据库

2. **领域覆盖限制**
   - 当前支持 13 个预定义领域
   - 超出范围的问题将使用通用推荐
   - 不支持动态领域扩展

3. **技能能力解析**
   - 技能能力（capabilities）未从 SKILL.md 实际解析
   - 当前为空列表
   - 相关性评分未考虑能力匹配

4. **依赖关系提取**
   - 使用简单关键字匹配（"requires" 或 "依赖"）
   - 可能遗漏复杂的依赖关系
   - 不支持间接依赖解析

5. **置信度估算**
   - 基于启发式算法
   - 可能与实际需求存在偏差
   - 不支持用户反馈调整

6. **执行时间预估**
   - 基于经验值
   - 实际执行时间可能有显著差异
   - 未考虑技能复杂度

7. **语言限制**
   - 当前仅支持中文关键词匹配
   - 不支持多语言问题描述

## 最佳实践建议

### 技能使用者

1. **提供清晰的描述**：详细的问题描述能提高推荐准确性
2. **指定领域（可选）**：已知领域时显式指定更准确
3. **检查依赖关系**：执行前检查技能依赖是否满足
4. **关注置信度**：低置信度（< 0.6）的推荐可能不够准确
5. **按顺序执行**：建议按推荐的执行顺序执行技能

### 技能开发者

1. **声明依赖关系**：在 SKILL.md 中清晰声明依赖
2. **使用标准关键字**：使用 "requires:" 或 "依赖:" 声明依赖
3. **定义能力**：在 SKILL.md 中明确定义技能能力
4. **使用已知领域**：使用支持的问题领域提高被推荐概率
