### 精炼描述
MCP-Aware Skill 生成器：强制单一职责、scope 严格控制、缺陷自省、建议拆分配套 Skill，输出到待应用-skill/

### 详细描述
本 Skill 作为元生成器，专门产生边界清晰、模块化的子 Skill。

**已识别的元 Skill 常见缺陷（横竖穷举后完善）**
横向维度：
- 容易 scope creep（用户一句话需求可能隐含 5+ 子任务）
- 穷举不完整（遗漏边缘输入/输出场景）
- 对 MCP 依赖判断偏保守或过度
- 生成的 Skill 之间缺乏显式接口约定
- token 消耗难以预测

纵向维度（层级）：
- 顶层：无法一次性构建完整系统（设计如此）
- 中层：生成的 Skill 之间协调依赖人工或额外 orchestrator Skill
- 底层：穷举逻辑本身依赖 LLM，可能引入偏差
- 运行时：无自我修复能力（符合要求，但仍是缺陷）

**已采取的补救措施**
- 强制在每个生成结果中声明局限 + 建议拆分 Skill 列表
- 重叠模式移到 overlaps/ 文件夹统一管理
- 底层约定设为只读 + 专用修复通道
- 生成的 Skill README 必须列出自身缺陷

适用于追求高度模块化、可维护 Skill 体系的场景。
**已识别元 Skill 缺陷（v1.2 更新）**  
横向：MCP 语义检测弱、缺少自动测试、无版本预览。  
纵向：顶层无闭环认知积累、中层模板静态、底层无持久记忆。

**自精炼机制（本次完善）**  
每次生成 Skill 时自动：
- 提取新重叠模式（结构化分类）
- 追加到 overlaps/common-overlaps.md
- 输出 cognition-update-proposal.md（可供未来修复 Skill 应用）
  从而让 generator 在“创造他人”的同时，持续完善自身对“结构化 Skill 世界”的认知。
